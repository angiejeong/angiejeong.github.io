Birthday Paradox Project

Motivation:
The Birthday Paradox states that in a group of just 23 randomly chosen people, there is a greater than 50% chance that at least two people share the same birthday. The paradox demonstrates that when looking for any match in a set, the likelihood of a collision is much higher than expected.

Functional requirements:
The program is separated into four main classes: Simulation.java, Experiment.java, Trial.java, and Birthday.java. The output of the program should be a table that will obviously change percentages each time you run the program, due to the randomness.
The top left of the program is Simulation.java, which creates an Experiment for the number of people from 5-100 at increments of 5 and with 20 trial each. Experiment.java class runs the experiment. Each experiment is defined by the number of people and the number of trials it has. When you run an experiment, you get the percentage of trials that had a repeat birthday. Trial.java represents a trial within an experiment, where each trial has the number of people in the trial. Finally, in the birthday class, Birthday.java creates a random month and day for a valid birthday.
